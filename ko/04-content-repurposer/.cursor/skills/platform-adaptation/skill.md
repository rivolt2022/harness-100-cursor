# Platform Adaptation — 플랫폼별 콘텐츠 적응 방법론

sns-copywriter와 blog-writer 에이전트가 콘텐츠를 변환할 때 활용하는 플랫폼 특화 지식.

## 왜 플랫폼 적응이 필요한가

같은 메시지를 모든 플랫폼에 복사-붙여넣기하면 **어느 곳에서도 최적이 아니다**. 각 플랫폼은 고유한 알고리즘, 포맷, 사용자 기대가 있다. 원본의 핵심을 유지하면서 형태를 완전히 바꾸는 것이 리퍼포징이다.

## 플랫폼별 콘텐츠 DNA

### X (트위터)
| 속성 | 값 |
|------|---|
| **핵심 가치** | 속도, 의견, 대화 |
| **최적 길이** | 단일 트윗 100~200자, 스레드 5~12개 |
| **알고리즘 우대** | 답글+리트윗 비율 높은 콘텐츠, 체류 시간 |
| **금기** | 외부 링크 단독 트윗 (도달 50% 감소), 이미지 없는 긴 텍스트 |
| **변환 전략** | 핵심 인사이트를 1트윗으로 → "왜 중요한가"를 스레드로 확장 |

**스레드 구조 공식**:
1. **훅 트윗**: 강렬한 주장 또는 반직관적 사실
2. **맥락 트윗**: "왜 이것이 중요한가"
3. **증거 트윗 3~5개**: 데이터, 사례, 인용
4. **전환 트윗**: "그래서 어떻게 해야 하나"
5. **CTA 트윗**: "이 스레드가 유용했다면 RT"
6. **셀프 리플라이**: 원본 링크

### Instagram
| 속성 | 값 |
|------|---|
| **핵심 가치** | 비주얼, 영감, 커뮤니티 |
| **최적 길이** | 캡션 125~150자 (접기 전), 카루셀 7~10장 |
| **알고리즘 우대** | 저장(Save), 공유(Share), 카루셀 완주율 |
| **금기** | 링크 텍스트 (클릭 불가), 텍스트만 있는 포스트 |
| **변환 전략** | 핵심 데이터를 시각화 카드로, 프로세스를 카루셀로 |

**카루셀 구조 공식**:
1. **표지 슬라이드**: 호기심 유발 제목 (큰 글씨)
2. **문제 정의**: "이런 경험 있으시죠?"
3. **해결책 3~5장**: 각 슬라이드 = 1개 포인트
4. **요약 슬라이드**: 핵심 3줄 정리
5. **CTA 슬라이드**: "저장하고 나중에 다시 보세요"

### LinkedIn
| 속성 | 값 |
|------|---|
| **핵심 가치** | 전문성, 경험, 네트워킹 |
| **최적 길이** | 1,300~2,000자 (3~4스크롤) |
| **알고리즘 우대** | 댓글 수, 체류 시간, 네이티브 콘텐츠 |
| **금기** | 외부 링크 본문 삽입 (도달 40% 감소), 해시태그 과다 |
| **변환 전략** | 개인 경험 + 전문 인사이트 조합, 링크는 첫 댓글에 |

**포스트 구조 공식**:
```
[훅: 첫 2줄에 스크롤을 멈추는 문장]

[빈 줄 — "더 보기" 위에 훅이 노출되도록]

[스토리/경험: 3~4문단]

[인사이트/교훈: 구체적이고 실행 가능한 것]

[CTA: 질문으로 끝내기 — 댓글 유도]

#해시태그 3~5개 (마지막 줄)
```

### 블로그 (SEO)
| 속성 | 값 |
|------|---|
| **핵심 가치** | 깊이, 검색 가능성, 상시 콘텐츠 |
| **최적 길이** | 1,500~3,000단어 |
| **SEO 우대** | H2/H3 구조, 키워드 밀도 1~2%, 내부/외부 링크 |
| **금기** | 키워드 스터핑, 얕은 콘텐츠, 중복 콘텐츠 |
| **변환 전략** | 원본을 확장+구조화, 검색 의도에 맞는 제목과 소제목 |

## 원본→플랫폼 변환 매트릭스

| 원본 요소 | X | Instagram | LinkedIn | 블로그 |
|----------|---|-----------|----------|-------|
| 핵심 주장 | 훅 트윗 | 표지 슬라이드 | 오프닝 2줄 | 제목(H1) |
| 데이터/통계 | 개별 트윗 | 인포그래픽 카드 | 인라인 인용 | 표/차트 |
| 사례/스토리 | 스레드 2~3개 | 카루셀 중간장 | 본문 스토리 | 섹션(H2) |
| 프로세스/방법 | 번호 스레드 | 스텝 카루셀 | 리스트 | 단계별 가이드 |
| 인용구 | 개별 트윗 | 텍스트 카드 | 들여쓰기 | 블록 인용 |
| CTA | 마지막 트윗 | 마지막 슬라이드 | 질문형 마무리 | CTA 버튼/배너 |

## 메시지 일관성 유지 체크리스트

플랫폼마다 형태는 달라도 **핵심 메시지는 동일**해야 한다:

- [ ] 모든 버전에서 같은 핵심 주장 1개가 명확히 전달되는가?
- [ ] 톤은 플랫폼에 맞게 조절했지만, 브랜드 보이스는 유지되는가?
- [ ] 데이터와 사실이 모든 버전에서 동일한가?
- [ ] CTA의 최종 목적지가 일관되는가?
- [ ] 시각적 요소(컬러, 폰트)가 브랜드 가이드를 따르는가?

# Scoring Optimizer — 정부과제 평가 고득점 전략

tech-writer, biz-writer의 고득점 작성 전략을 강화하는 스킬.

## 대상 에이전트

- **tech-writer** — 기술성 평가 고득점 전략을 적용한다
- **biz-writer** — 사업성 평가 고득점 전략을 적용한다
- **submission-reviewer** — 평가 기준 대비 완성도를 검증한다

## 평가 항목별 고득점 전략

### 기술성 평가 (일반 40-50점)

#### 기술 개발 목표 및 내용 (15-20점)

```
고득점 전략:
1. 개발 목표를 정량 지표로 제시
   BAD: "성능을 향상시킨다"
   GOOD: "처리 속도 기존 대비 300% 향상 (100ms → 30ms)"

2. 기존 기술 대비 차별성 3가지 이상 제시
   - 성능 차별성 (속도, 정확도)
   - 방법론 차별성 (새로운 접근)
   - 적용 범위 차별성 (확장성)

3. 핵심 기술 요소를 도식화
   - 시스템 아키텍처 다이어그램
   - 기술 로드맵 (단계별)
   - 알고리즘 흐름도

4. 특허/논문 분석으로 기술 트렌드 제시
```

#### 추진 체계 및 일정 (10-15점)

```
고득점 전략:
1. WBS (Work Breakdown Structure) 상세화
   - 단계 → 세부과제 → 세세부과제 3단계
   - 마일스톤 명확 (Go/No-Go 기준)

2. 간트차트 필수 포함
   - 월별 진도 계획
   - 의존 관계 표시
   - 크리티컬 패스 식별

3. 인력 투입 계획
   - 역할별 M/M (Man-Month)
   - 역량 매칭 (학위, 경력, 프로젝트)

4. 위험 관리 계획
   - 기술적 리스크 3가지 + 대응 방안
   - 일정 리스크 + 버퍼 계획
```

### 사업성 평가 (일반 30-40점)

#### 시장성 (10-15점)

```
고득점 전략:
1. 시장 규모 이중 검증 (상향식 + 하향식)
2. 국내+해외 시장 모두 기술
3. CAGR + 성장 동인 3가지
4. 목표 시장 세그먼트 구체화
5. 정부 정책 연계 (정책 키워드 반영)
```

#### 사업화 전략 (10-15점)

```
고득점 전략:
1. 사업화 로드맵 (기술개발 → 인허가 → 양산 → 판매)
2. 매출 계획 (3-5년, 보수적 시나리오 포함)
3. 판로 확보 방안 구체화 (LOI, 시범사업)
4. 수익 모델 + 가격 전략
5. 지식재산권 확보 계획 (특허, 인증)
```

## 감점 방지 체크리스트

```
[ ] 평가 항목별 페이지 수가 배점에 비례하는가?
[ ] 모든 평가 항목에 대응하는 내용이 있는가?
[ ] 정량적 수치에 출처가 명시되어 있는가?
[ ] 도표/그림이 적절히 포함되어 있는가? (3-5개/10페이지)
[ ] 오탈자/문법 오류가 없는가?
[ ] 공고문의 핵심 키워드가 반영되어 있는가?
[ ] 제출 양식을 정확히 따르고 있는가?
[ ] 페이지 제한을 초과하지 않는가?
[ ] 참고문헌이 최근 3년 이내인가?
[ ] 연구 윤리/보안 관련 사항이 기재되어 있는가?
```

## 공고문 키워드 분석 방법

```
1. 공고문에서 반복되는 키워드 추출
2. 평가 기준에 사용된 형용사/부사 수집
   예: "혁신적", "차별화된", "구체적인"
3. 정책 키워드 매핑
   예: "디지털 전환", "탄소중립", "초격차"
4. 사업계획서에 해당 키워드를 자연스럽게 반영
5. 키워드 출현 빈도 체크 (최소 3회 이상)
```

## 발표 평가 대비

```
시간 배분 (10분 발표 기준):
  1분: 기업/팀 소개
  2분: 문제 정의 + 기술 개발 목표
  3분: 핵심 기술 + 차별성
  2분: 시장성 + 사업화 전략
  1분: 추진 일정 + 기대 효과
  1분: 마무리 + Q&A 준비

Q&A 대비 TOP 5:
  1. "기존 기술과 뭐가 다른가요?"
  2. "시장 규모 근거가 무엇인가요?"
  3. "팀에 부족한 역량은 어떻게 보완하나요?"
  4. "예산 이 항목이 왜 필요한가요?"
  5. "사업화 일정이 현실적인가요?"
```

# Price Elasticity Calculator — 가격 탄력성 분석 + 최적 가격

pricing-simulator와 competitive-analyst의 가격 분석을 강화하는 스킬.

## 대상 에이전트

- **pricing-simulator** — 가격 변경 시나리오의 영향을 정량 분석한다
- **competitive-analyst** — 경쟁사 가격 변동의 교차 탄력성을 분석한다

## 가격 탄력성 기본 공식

### 수요의 가격 탄력성 (PED)

```
PED = (수요량 변화율) / (가격 변화율)
    = (ΔQ/Q) / (ΔP/P)

중간점 공식 (더 정확):
PED = [(Q2-Q1)/((Q1+Q2)/2)] / [(P2-P1)/((P1+P2)/2)]

해석:
  |PED| > 1: 탄력적 — 가격 변화에 수요 민감
  |PED| = 1: 단위탄력적 — 매출 변화 없음
  |PED| < 1: 비탄력적 — 가격 변화에 수요 둔감

예시:
  가격 10% 인상, 수요 15% 감소
  PED = -15% / 10% = -1.5 (탄력적)
```

### 교차 탄력성 (Cross-Price Elasticity)

```
XED = (자사 수요량 변화율) / (경쟁사 가격 변화율)

해석:
  XED > 0: 대체재 관계 (경쟁사 가격↑ → 자사 수요↑)
  XED < 0: 보완재 관계
  XED ≈ 0: 무관

예시:
  경쟁사 가격 20% 인상, 자사 수요 10% 증가
  XED = 10% / 20% = 0.5 (약한 대체재)
```

## 산업별 탄력성 벤치마크

| 산업/제품 | PED 범위 | 특성 |
|----------|---------|------|
| SaaS (기업용) | -0.5 ~ -1.0 | 비탄력적 (전환비용 높음) |
| SaaS (개인용) | -1.0 ~ -2.0 | 탄력적 (대안 많음) |
| 이커머스 (일반) | -1.5 ~ -2.5 | 탄력적 (가격비교 용이) |
| 이커머스 (럭셔리) | -0.5 ~ -1.0 | 비탄력적 (브랜드 충성) |
| 식료품 (필수) | -0.1 ~ -0.5 | 매우 비탄력적 |
| 외식 | -1.0 ~ -2.0 | 탄력적 |
| 모바일 앱 | -2.0 ~ -3.0 | 매우 탄력적 |

## 최적 가격 도출

### 수익 최대화 가격

```
수익 R = P × Q(P)
수요 함수: Q(P) = a - b×P  (선형 가정)

최적 가격 P* = a / (2b)

예시:
  Q = 10,000 - 50P (월간 수요)
  P* = 10,000 / (2×50) = 100 (원)
  Q* = 10,000 - 50×100 = 5,000
  R* = 100 × 5,000 = 500,000
```

### 이익 최대화 가격

```
이익 π = (P - C) × Q(P)
  C = 단위 변동비

최적 가격 P* = (a + b×C) / (2b)

예시:
  Q = 10,000 - 50P, 단위 변동비 C = 30
  P* = (10,000 + 50×30) / (2×50) = 115
  Q* = 10,000 - 50×115 = 4,250
  π* = (115-30) × 4,250 = 361,250
```

## 가격 변경 시뮬레이션

### 시나리오 분석 템플릿

```
현재 상태:
  가격: P₀ = 10,000원
  수요: Q₀ = 5,000건/월
  매출: R₀ = 5,000만원/월
  탄력성: PED = -1.5

시나리오 1: 10% 인상
  새 가격: 11,000원
  수요 변화: -15% → 4,250건
  새 매출: 4,675만원 (-6.5%)
  순이익 변화: [마진 고려 계산]

시나리오 2: 10% 인하
  새 가격: 9,000원
  수요 변화: +15% → 5,750건
  새 매출: 5,175만원 (+3.5%)
  순이익 변화: [마진 고려 계산]

시나리오 3: 20% 인상 + 기능 추가
  새 가격: 12,000원
  예상 탄력성: -1.0 (가치 증가로 둔화)
  수요 변화: -20% × (-1.0/−1.5) = -13.3%
  ...
```

## 가격 인상 실행 전략

```
1. 단계적 인상 (Gradual)
   - 연 2-3회 소폭 인상 (5-10%)
   - 고객 저항 최소화
   - 기존 고객 할인 유지 기간 설정

2. 가치 기반 인상 (Value-based)
   - 새 기능 추가와 동시에 인상
   - "더 많은 가치를 제공합니다" 메시지
   - 기존 플랜 유지 + 프리미엄 플랜 추가

3. 세분화 인상 (Segmented)
   - 신규 고객부터 적용
   - 기존 고객 6-12개월 유예
   - 가격 민감 세그먼트 별도 관리

4. 할인 축소 (Discount reduction)
   - 정가 유지, 할인 폭 축소
   - 인상이 아닌 "할인 정상화"
```

## 가격 전쟁 대응

```
경쟁사 가격 인하 시 의사결정 트리:

1. 교차 탄력성 높음 (XED > 0.5)?
   ├── Yes → 고객 이탈 리스크 높음
   │   ├── 우리가 원가 우위? → 가격 매칭
   │   └── 아닌 경우 → 가치 차별화 강화
   └── No → 무시 가능, 가격 유지

2. 경쟁사 인하가 지속 가능한가?
   ├── 일시적 (프로모션) → 무시 또는 한정 프로모션
   └── 구조적 (비용 우위) → 가격 외 경쟁 전략
```

# Decision Trigger Mapper — 의사결정 트리거 맵 설계 도구

strategy-architect 에이전트의 전략 실행 설계 역량을 강화하는 전문 스킬.

## 적용 대상 에이전트

- **strategy-architect** — 대응 전략 수립, 의사결정 트리거 설계

## 전략 유형 분류 체계

### 3-Tier 전략 포트폴리오

| 유형 | 정의 | 실행 조건 | 투자 수준 |
|------|------|----------|----------|
| **로버스트 전략** | 모든 시나리오에서 유효 | 즉시 실행 | 전액 투자 |
| **헤지 전략** | 대부분 시나리오에서 유효 | 조기 착수, 방향 조정 | 중간 투자 |
| **옵션 전략** | 특정 시나리오에서만 유효 | 트리거 발동 시 실행 | 최소 투자(옵션 확보) |

### 전략 유형 판별 의사결정 트리

```
이 전략이 4개 시나리오 모두에서 가치 있는가?
├── YES → 로버스트 전략 (Core)
│         → 즉시 실행, 자원 집중
└── NO
    ├── 2-3개에서 가치 있는가?
    │   ├── YES → 헤지 전략 (Hedge)
    │   │         → 초기 투자, 피벗 가능 설계
    │   └── NO → 1개에서만 가치 있는가?
    │       ├── YES & 영향 크면 → 옵션 전략 (Option)
    │       │                   → 최소 투자, 트리거 대기
    │       └── YES & 영향 작으면 → 보류
    └── 어떤 시나리오에서도 불확실 → 재분석 필요
```

## 의사결정 트리거 맵 설계

### 트리거 카드 템플릿

```markdown
### 트리거 #N: [트리거명]

**관련 전략**: [어떤 옵션 전략을 활성화하는가]
**관련 시나리오**: [어떤 시나리오로 이동 시 발동하는가]

| 항목 | 내용 |
|------|------|
| 관찰 지표 | [무엇을 모니터링하는가] |
| 데이터 소스 | [어디서 데이터를 수집하는가] |
| 임계값 | [어떤 수준에서 발동하는가] |
| 확인 기간 | [얼마나 지속되어야 트리거 확정인가] |
| 대응 행동 | [구체적으로 무엇을 하는가] |
| 실행 책임 | [누가 의사결정하는가] |
| 소요 자원 | [필요 예산/인력/시간] |
| 철회 조건 | [언제 트리거를 해제하는가] |
```

### 트리거 타입

| 타입 | 설명 | 예시 |
|------|------|------|
| 레드라인 트리거 | 넘으면 즉시 행동 | 시장점유율 < 15% |
| 누적 트리거 | 여러 신호가 쌓이면 행동 | 3개 이상 경보 신호 동시 발동 |
| 시간 트리거 | 특정 시점에 평가 | 매 분기 전략 리뷰 |
| 이벤트 트리거 | 특정 사건 발생 시 행동 | 경쟁사 M&A 발표 |

## 리얼옵션 사고 프레임워크

전략적 옵션을 금융 옵션 비유로 설계한다:

| 금융 옵션 | 전략적 옵션 대응 | 예시 |
|----------|----------------|------|
| 콜옵션 (매수 권리) | 성장 옵션 | 신시장 진출 준비 (소규모 테스트) |
| 풋옵션 (매도 권리) | 축소/철수 옵션 | 사업부 매각 조건 정의 |
| 스트래들 | 양방향 옵션 | 양쪽 시나리오 모두 대비 |
| 프리미엄 | 옵션 유지 비용 | 파일럿 프로젝트 비용 |
| 행사가 | 트리거 임계값 | 시장 규모 도달 기준 |
| 만기 | 결정 기한 | 전략 리뷰 시점 |

### 옵션 가치 평가 기준

| 기준 | 질문 | 높으면 옵션 가치↑ |
|------|------|-----------------|
| 불확실성 | 해당 영역의 불확실성이 큰가? | 불확실할수록 옵션 가치 ↑ |
| 비가역성 | 한번 실행하면 돌이킬 수 없는가? | 비가역적일수록 옵션 가치 ↑ |
| 학습 가능성 | 시간이 지나면 정보가 늘어나는가? | 학습 가능할수록 옵션 가치 ↑ |
| 옵션 비용 | 옵션 유지 비용이 감당 가능한가? | 비용 낮을수록 실행 용이 |

## 실행 로드맵 통합

### 시간축별 행동 분류

| 시간축 | 행동 유형 | 예산 비중 |
|-------|----------|----------|
| 즉시 (0-3개월) | 로버스트 전략 착수 + 모니터링 체계 구축 | 50% |
| 단기 (3-12개월) | 헤지 전략 초기 실행 + 옵션 확보 | 30% |
| 중기 (1-3년) | 트리거 기반 옵션 행사/포기 | 15% |
| 장기 (3-5년) | 전략 포트폴리오 재평가 | 5% |

### 전략 리뷰 주기

```
분기 리뷰: 트리거 모니터링 → 신호 평가 → 옵션 행사 여부 결정
반기 리뷰: 시나리오 유효성 재검증 → 새로운 변수 추가 → 매트릭스 업데이트
연간 리뷰: 풀 시나리오 갱신 → 전략 포트폴리오 재구성
```

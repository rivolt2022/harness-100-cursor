# Win Theme Builder — 제안 Win Theme 구축 도구

capability-matcher, proposal-reviewer 에이전트의 차별화 전략 수립 역량을 강화하는 전문 스킬.

## 적용 대상 에이전트

- **capability-matcher** — 실적 매칭, Win Theme 도출
- **proposal-reviewer** — Win Theme 일관성 검증, 차별화 점검

## Win Theme 도출 프로세스

### Step 1: 고객 핫 버튼(Hot Button) 분석

RFP 이면의 진짜 관심사를 파악한다:

| 소스 | 분석 포인트 |
|------|-----------|
| 평가 배점표 | 고배점 항목 = 고객의 최우선 관심사 |
| RFP 반복 키워드 | 3회 이상 등장하는 단어/구문 |
| 제약 조건 | "반드시", "필수" 등 강조 표현 |
| 배경 설명 | 현재 Pain Point 추론 |
| 제출 요구 | 상세히 요구하는 영역 = 중점 평가 |

### Step 2: Win Theme 매트릭스

```markdown
| Hot Button | 우리 강점 | 경쟁사 약점 | Win Theme |
|-----------|----------|-----------|-----------|
| 안정적 운영 | 99.9% SLA 달성 이력 | 장애 이력 있음 | "무중단 운영의 검증된 파트너" |
| 비용 효율 | 클라우드 최적화 경험 | 온프레미스 중심 | "30% TCO 절감의 클라우드 전문가" |
| 기술 역량 | 관련 인증 10건+ | 미보유 | "국내 최다 인증 보유 전문 조직" |
```

### Step 3: Win Theme 정제 기준

좋은 Win Theme의 요건:
- **고객 관점**: 고객이 얻는 가치 중심 (우리 자랑 X)
- **차별적**: 경쟁사가 동일하게 주장할 수 없는 것
- **증명 가능**: 실적, 데이터, 인증으로 뒷받침
- **간결**: 1문장으로 표현 가능
- **반복 가능**: 제안서 전체에 일관되게 관통

### Win Theme 문장 구조

```
[정량 근거] + [차별화 행동] + [고객 가치]

예시:
"20건의 공공 SI 실적으로 검증된 방법론으로, 납기 준수율 100%를 보장합니다"
"국내 유일 ISO 27001 + ISMS-P 동시 인증 기업으로, 최고 수준의 보안을 제공합니다"
```

## Ghost Team 분석 (경쟁사 예상 전략)

### 경쟁사 강약점 매트릭스

```markdown
| 평가 항목 | 자사 | 경쟁사A | 경쟁사B | 전략 |
|----------|------|--------|--------|------|
| 유사 실적 | 15건 | 20건 | 8건 | B 약점 공략 |
| 기술 인증 | 10개 | 5개 | 12개 | 차별점 강조 |
| 핵심 인력 | 7명 | 10명 | 5명 | 질적 우위 강조 |
| 가격 | 중 | 고 | 저 | 가치 제안 |
```

### Counter 전략

| 경쟁사 예상 주장 | 우리의 Counter |
|----------------|--------------|
| "더 큰 실적 보유" | "규모보다 유사성: 동일 업종/규모 실적 N건" |
| "더 저렴한 가격" | "TCO 분석: 운영비 포함 시 총비용 절감" |
| "더 많은 인력" | "핵심 인력의 질: 평균 경력 N년, 인증 M개" |

## 제안서 Win Theme 관통 전략

### 섹션별 Win Theme 반영 가이드

| 제안서 섹션 | Win Theme 반영 방식 |
|-----------|-------------------|
| Executive Summary | 3개 Win Theme 직접 선언 |
| 기술 제안 | 방법론/아키텍처에 Win Theme 근거 내장 |
| 수행 체계 | 인력/조직에 Win Theme 증거 배치 |
| 유사 실적 | Win Theme 증명 사례로 실적 선별 |
| 가격 제안 | 가격 정당화에 Win Theme 연결 |
| 차별화 | Win Theme 종합 + 정량 비교표 |

### 일관성 검증 체크리스트 (proposal-reviewer용)

- [ ] Executive Summary에 Win Theme 3개가 명시되어 있는가?
- [ ] 각 Win Theme이 본문에서 최소 3회 이상 근거와 함께 등장하는가?
- [ ] 실적 섹션이 Win Theme을 증명하는 사례 중심으로 선별되었는가?
- [ ] 가격 섹션이 Win Theme의 가치와 연결되어 있는가?
- [ ] 경쟁사 대비 차별점이 정량적으로 비교되어 있는가?

# Rhetoric Patterns — 수사학 패턴 라이브러리

speech-writer / debate-preparer 에이전트의 설득력을 높이는 수사학 패턴.

## 연설 구조 패턴

### 패턴 1: Monroe's Motivated Sequence

| 단계 | 목적 | 기법 |
|------|------|------|
| 1. Attention (주의) | 관심 집중 | 충격 통계, 스토리, 질문 |
| 2. Need (필요) | 문제 인식 | 현실 직시, 데이터 제시 |
| 3. Satisfaction (해결) | 해결책 제시 | 구체적 방안, 논리적 설명 |
| 4. Visualization (시각화) | 미래 상상 | 성공/실패 시나리오 |
| 5. Action (행동) | 행동 유도 | 구체적 첫 걸음, CTA |

**적합**: 설득/제안 발표, 투자 피치, 정책 연설

### 패턴 2: PREP (Point-Reason-Example-Point)

```
Point: "우리는 클라우드 마이그레이션을 올해 완료해야 합니다"
Reason: "현재 온프레미스 비용이 연간 30% 증가하고 있기 때문입니다"
Example: "경쟁사 A는 마이그레이션 후 인프라 비용을 40% 절감했습니다"
Point: "따라서 올해 3분기까지 클라우드 전환을 완료해야 합니다"
```

**적합**: 짧은 발언, Q&A 답변, 회의 발언

### 패턴 3: 문제-해결-이익 (PSB)

```
Problem: 현재 [문제] 때문에 [부정적 결과]가 발생하고 있습니다
Solution: [제안]을 통해 이 문제를 해결할 수 있습니다
Benefit: 이를 통해 [구체적 이익]을 얻을 수 있습니다
```

### 패턴 4: 스토리텔링 아크

```
1. 일상 (상황 설정)
2. 도전 (갈등/문제)
3. 전환점 (인사이트/결정)
4. 변화 (행동/결과)
5. 교훈 (메시지)
```

## 아리스토텔레스 3요소

| 요소 | 설명 | 적용 기법 |
|------|------|----------|
| **Ethos** (신뢰) | 화자의 자격/신뢰도 | 경력 언급, 전문성, 공정성 |
| **Pathos** (감정) | 청중의 감정 동조 | 스토리, 비유, 시각 자료 |
| **Logos** (논리) | 논리적 근거 | 데이터, 통계, 인과관계 |

### 발표 유형별 비중

| 발표 유형 | Ethos | Pathos | Logos |
|----------|-------|--------|-------|
| 기조연설 | 30% | 40% | 30% |
| 투자 피치 | 20% | 20% | 60% |
| 동기부여 | 20% | 50% | 30% |
| 학술 발표 | 30% | 10% | 60% |
| 위기 대응 | 40% | 30% | 30% |

## 논증 구조 (토론용)

### Toulmin 모델

```
Claim (주장): [결론/입장]
  ← Data (근거): [사실/증거]
  ← Warrant (보장): [근거→주장 연결 논리]
  ← Backing (뒷받침): [보장의 근거]
  ← Qualifier (한정): "대부분의 경우", "일반적으로"
  ← Rebuttal (반론): "단, [예외 상황]에서는..."
```

### 반론 대응 전략

| 전략 | 설명 | 예시 |
|------|------|------|
| 수용 후 전환 | 일부 인정 후 핵심 재강조 | "맞습니다, 하지만 더 중요한 것은..." |
| 리프레이밍 | 관점 전환 | "비용이 아닌 투자로 봐야 합니다" |
| 증거 대응 | 반대 증거 제시 | "실제 데이터를 보면..." |
| 조건부 수용 | 제한적 인정 | "특정 상황에서는 맞지만, 우리 맥락은..." |

## 수사적 장치

| 장치 | 설명 | 예시 |
|------|------|------|
| 반복 (Anaphora) | 문장 시작 반복 | "우리는 할 수 있습니다. 우리는 해야 합니다." |
| 대조 (Antithesis) | 대비 구조 | "작게 생각하지 마세요, 크게 꿈꾸세요" |
| 삼분법 (Rule of Three) | 3개씩 나열 | "빠르고, 정확하고, 안전합니다" |
| 수사 질문 (Rhetorical Q) | 답 알려진 질문 | "이것이 정말 최선입니까?" |
| 비유 (Analogy) | 익숙한 것에 비유 | "데이터는 새 시대의 석유입니다" |
| 숫자 활용 | 구체적 수치 | "3초마다 1건의 사이버 공격이..." |

## 품질 체크리스트
